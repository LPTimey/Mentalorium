\section{Definitions}
\subsection{Participants}

We define the following sets:
\[
\begin{aligned}
\mathrm{Groups}\ G &:= \{\mathrm{Active}, \mathrm{Control}\},\\
\mathrm{Case\ Vignettes}\ C &:= \{\mathrm{Amina}, \mathrm{Aylin}, \mathrm{Lena},
        \mathrm{Martina}, \mathrm{Mathias}\},\\
\mathrm{Truthfulness}\ T &:= \{\mathrm{Truth}, \mathrm{Untruth}\}.
\end{aligned}
\]

A \emph{round} is a pair consisting of a case vignette and a
truthfulness label. Hence, the set of all rounds is
\[
R := C \times T.
\]

A \emph{session} consists of exactly three rounds.
Formally, a session is an ordered triple
\[
s = (r_1,r_2,r_3) \in R^3.
\]

However, not every element of \(R^3\) is considered valid.
A valid session must satisfy the following constraints:
\[
\left|
\left\{
i \in \{1,2,3\}
\mid
r_i = (c,\mathrm{Truth})
\text{ for some } c \in C
\right\}
\right|
= 1,
\]

i.e.\ exactly one round in the session is labelled
\(\mathrm{Truth}\).

Furthermore,
\[
\forall i,j \in \{1,2,3\}:
\quad
i \neq j
\Rightarrow
c_i \neq c_j,
\]
where \(r_i = (c_i,t_i)\).
That is, no case vignette may occur more than once within the same
session.

We denote the set of all valid sessions by
\[
S \subseteq R^3.
\]

A participant is a triple
\[
p = (g,s_1,s_2) \in G \times S \times S,
\]
where
\[
g \in G,
\qquad
s_1,s_2 \in S.
\]

The two sessions of a participant must share exactly one case
vignette. Formally,
\[
\left|
\left\{
c \in C
\mid
\exists\, t_1,t_2 \in T:
(c,t_1) \in \{r_1,r_2,r_3\}
\land
(c,t_2) \in \{r'_1,r'_2,r'_3\}
\right\}
\right|
= 1,
\]
where
\[
s_1 = (r_1,r_2,r_3),
\qquad
s_2 = (r'_1,r'_2,r'_3).
\]

Thus:
\[
\mathrm{Participants}\ P := G \times S \times S
\]
where both $S$ fulfill all previous conditions

An example of a valid participant is
\[
\begin{aligned}
p = (
&\mathrm{Active},\\[2pt]
&((\mathrm{Amina},\mathrm{Truth}),
  (\mathrm{Lena},\mathrm{Untruth}),
  (\mathrm{Mathias},\mathrm{Untruth})),\\[2pt]
&((\mathrm{Martina},\mathrm{Truth}),
  (\mathrm{Aylin},\mathrm{Untruth}),
  (\mathrm{Mathias},\mathrm{Untruth}))
).
\end{aligned}
\]
\subsection{Participant Allocation}

To generate participants satisfying the constraints defined in the
previous subsection, we employ a deterministic pseudo-random allocation
algorithm, denoted by \(A_s\) (Participant Allocator).

For a fixed pseudo-random seed \(s\), the algorithm defines a mapping

\[
A_s : \{0,\dots,N-1\} \to P
\]

where

\[
A_s(n) = p_n,
\]

with

\begin{itemize}
    \item \(P\) denoting the set of valid participants defined previously,
    \item \(n \in \mathbb{N}_0\) denoting the participant index,
    \item \(p_n \in P\) denoting the generated participant corresponding
    to index \(n\),
    \item \(s \in \mathbb{N}\) denoting the global pseudo-random seed.
\end{itemize}

In the present study, the seed value
\(
s = 403
\)
was used.
The algorithm is deterministic with respect to the seed \(s\),
thereby ensuring reproducibility of the participant allocation.

The reference Python implementation is provided in the attached file
\textattachfile[color=0.5 0.5 0.5]{allocate.py}{allocate.py}.

\subsection{Automation bias}

For the purposes of this study, we define a heuristic automation-bias 
score $B$ that serves as an operational measure of automation bias.
$B$ is modelled as a continuous function that maps
evaluation parameters to a scalar value in the interval $[0,1]$,
representing the degree of evidence that automation bias has occurred.

The function is defined as:

\[
B : \{0,1\} \times \{1,2,3,4,5\} \times [0,100] \to [0,1],
\]
\[
B(i,r,c),
\]

where:
\begin{itemize}
    \item $B(i,r,c) \in [0,1]$ denotes the automation bias score,
    \item $i \in \{0,1\}$ indicates correctness of the automated answer
          ($1$ = correct, $0$ = incorrect),
    \item $r \in \{1,2,3,4,5\}$ denotes the evaluator's rating of the answer
          (1 = very bad, 5 = very good),
    \item $c \in [0,100]$ denotes the evaluator's confidence in the rating.
\end{itemize}

\subsubsection{Definition of the Bias Function}

The function is defined as:

\[
B(i,r,c)
=
(1 - i)
\cdot
\frac{r - 1}{5 - 1}
\cdot
\frac{c}{100}.
\]

\subsubsection{Interpretation}

The term $(1 - i)$ ensures that automation bias is only considered when the
automated answer is incorrect. If the answer is correct ($i=1$), then
$B(i,r,c)=0$ regardless of the other parameters.

The term
\(
\frac{r - 1}{4}
\)
maps the rating to a normalized interval $[0,1]$, where higher ratings
indicate stronger perceived quality of the answer. In the context of
automation bias, higher ratings for incorrect answers indicate a stronger
tendency toward bias.

The term
\(
\frac{c}{100}
\)
represents the evaluator's confidence in their rating. Higher confidence
strengthens the evidence of automation bias.

By construction, \(B(i,r,c) \in [0,1]\).

\subsubsection{Threshold for Minimal Automation Bias}

The threshold value of $0.25$ was selected a priori as an operational cutoff corresponding to a neutral evaluation ($r=3$) combined with moderate confidence ($c=50$) of an incorrect recommendation.
This value was resolved like this:

\[
(i,r,c) = (0,3,50).
\]

Substituting into the function yields:

\[
\begin{aligned}
B(0,3,50)
&= (1 - 0)
\cdot \frac{3 - 1}{4}
\cdot \frac{50}{100} \\[4pt]
&= 1 \cdot \frac{2}{4} \cdot 0.5 \\[4pt]
&= 1 \cdot 0.5 \cdot 0.5 \\[4pt]
&= 0.25.
\end{aligned}
\]

We therefore define the minimal automation bias threshold as:

\[
B_{\text{threshold}} = 0.25,
\quad \text{and we consider}
\quad
B(i,r,c) > 0.25
\]
as indicating evidence of automation bias.

\section{Methodology}
\subsection{General}
Our tests followed this formula:

\begin{enumerate}
    \item Participant gives consent
    \item Participant is given a short context description \\
    \textit{"Wir haben eine App für KI-gestützte mentale Gesundheit vorbereitet.
            Sie kann Patientenfälle einschätzen und Behandlungsvorschläge geben."}
    \item Participant gives themselves a nickname
    \item Participant answers demographics questions
    \begin{itemize}
        \item Gender
        \begin{itemize}
            \item Male
            \item Female
            \item Nonbinary
            \item Other
            \item Prefer not to say
        \end{itemize}
        \item Age
    \end{itemize}
    \item Question About AI
    \begin{itemize}
        \item How do you view the use of Artificial Intelligence in terms of usefulness?\\
        Likert Scale \(\{1,2,3,4,5\} \in\) Not useful at all $\leftrightarrow$ very useful
        \item How do you view the use of Artificial Intelligence in terms of acceptability?\\
        Likert Scale \(\{1,2,3,4,5\} \in\) Very unacceptable $\leftrightarrow$ very acceptable
        \item How do you view the use of Artificial Intelligence in the field of mental health in terms of usefulness?\\
        Likert Scale \(\{1,2,3,4,5\} \in\) Not useful at all $\leftrightarrow$ very useful
        \item How do you view the use of Artificial Intelligence in the field of mental health in terms of acceptability?\\
        Likert Scale \(\{1,2,3,4,5\} \in\) Very unacceptable $\leftrightarrow$ very acceptable
        \item How frequently do you use Artificial Intelligence in the field of mental health?\\
        Likert Scale \(\{1,2,3,4,5\} \in\) Not at all $\leftrightarrow$ very often
        \item How frequently do you use Artificial Intelligence otherwise?\\
        Likert Scale \(\{1,2,3,4,5\} \in\) Not at all $\leftrightarrow$ very often
    \end{itemize}
    %% this is (6)
    \item Participant is handed Case Vignette \& Instructor sets up App and enters values into database %% <- this is (6)
    \item participant reads case vignette
    \item participant signals to continue
    \item participant gets App
    \item (If Active) participant gets Disclaimer
    \item (If Active) participant waits out and reads Disclaimer
    \item participant sends Prompt
    \item (If Active) participant gets inline Disclaimer
    \item participant reads Answer
    \item participant answers questions
    \begin{itemize}
        \item How do you rate the AI Answer?\\
        Likert Scale \(\{1,2,3,4,5\} \in\) very bad $\leftrightarrow$ very good
        \item How certain are you in your choice?\\
        \([0,100]\)
        \item questions from UEQ+
        \begin{itemize}
            \item Intuitive Use
            \item Trustworthiness of Content
            \item Response behavior
            \item Response quality
            \item Clarity
            \item Risk Handling
        \end{itemize}
    \end{itemize}
    \item Steps 6–16 were repeated two additional times using different case vignettes.
    \item end of study Session
    \item 1 week passes
    \item repeat (1)
    \item remind of context
    \item patient remembers nickname or is told if they forgot
    \item repeat (5) to (16) but always without disclaimers
\end{enumerate}

\section{Hypotheses}
We conducted our study under the following hypotheses:

\begin{description}
\item[$H_1$] Exposure to a design intervention in a mental health AI application leads to a sustained reduction in automation bias, such that participants in the intervention group also exhibit a lower automation bias scores than participants in the control group during a subsequent usage session in which no additional intervention is provided.

\item[$H_0$] There are no differences between the intervention and control groups in terms of changes in automation bias over time. The design intervention does not lead to a long-term effect on automation bias.
\end{description}
